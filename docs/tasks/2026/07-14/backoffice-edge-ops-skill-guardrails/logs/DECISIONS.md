- 네 guardrail은 UI 설계, 코드 구조, 문서 패키지 규칙에 걸쳐 있고 Project Legibility는 generated snapshot을 조립한다.
- 운영 판단과 primitive 재사용은 design-user-interfaces, 최소 경계의 의미 보존은 structure-first, 문서 소유권의 runtime UX 비규정은 tighten-docs가 각각 소유한다.
- 각 문장을 오독이 발생한 규칙 바로 옆에 두면서 source-owner-audit, purpose-fit-design과 조립 문서의 기존 역할을 중복하지 않는다.
- 세 canonical skill의 영문 정본과 한국어 페어만 수정하고 Project Legibility는 push된 commit을 lock한 generated 결과만 갱신한다.
