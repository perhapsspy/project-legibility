# 결정 기록

**2026-07-11**

- **Background:** 일반 설명에 현재 스킬·저장소 개수를 반복하면 구성이 바뀔 때 여러 문서가 쉽게 어긋난다.
- **Decision:** 일반 설명에는 개수를 고정해 적지 않고 README의 포함 스킬 목록과 `sources.lock.json`이 실제 구성을 소유하게 한다.
- **Why:** 구성 변경 때 수정해야 할 사용자 문서의 수를 줄이고 현재 정보의 소유자를 분명히 할 수 있다.
- **Impact:** 공개 문구는 수량보다 사용자가 얻는 결과와 canonical 경로를 설명한다.
