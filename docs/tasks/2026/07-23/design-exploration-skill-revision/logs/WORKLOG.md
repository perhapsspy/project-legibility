# Worklog

**2026-07-23**

- 현재 0.4.1 플러그인 묶음, 제품 계약, 정본 방향 문서와 스킬 선택 검증 범위를 점검했다. 제품 계약 한·영 문서에 이상 징후와 경험칙의 조사 단서 성격, 조건부 탐색과 명백한 실패 경계를 추가했다. `purpose-fit-design`은 초기 방향 탐색, `structure-first`는 코드를 이해·추적·검증하기 어려운 구체적인 문제에 선택하는 역할로 조정했다.
- 샌드박스에서 기준안과 후보안을 가린 세 비교 시나리오를 별도 에이전트가 실행하고 독립 검토자가 비교했다. 후보안은 세 사례 모두 전체적으로 우세했고 명백한 실패는 없었다. 국소 조건식 사례에서는 후보안도 구조를 늘리지 않았지만 표현을 불필요하게 바꾼 작은 회귀가 있었다. 이를 반영해 기존 단위가 이미 일관된 변경을 수용하면 `structure-first`를 선택하지 않는 검증 사례를 추가했다. 이벤트 기반 사례에서는 기준안이 완료·실패 처리를 범용 도우미로 중앙화한 반면, 후보안은 종료 상태를 판정하는 조건만 분리하고 각 생명주기의 책임을 유지했다.
- 독립 설계 검토에서 단일 소유자를 강제하는 문구와 지나치게 넓은 결정 중단 조건을 발견했다. 이를 서로 경쟁하지 않는 결정 경로와 사용자 권한·되돌리기 어려운 약속에만 적용되는 중단 조건으로 좁혔다. 의도적인 다중 쓰기 규약과 작고 되돌릴 수 있는 기존 의존성 사용 대조군에서 각각 중앙화와 불필요한 중단이 생기지 않음을 확인했다.
- `purpose-fit-design`과 `structure-first`의 동작 변경을 먼저 정본에 올린 뒤, 세 딥리즈너가 제품 문서·목적 탐색 스킬·구조 탐색 스킬의 한국어를 나누어 검토했다. 일반 개념은 자연스러운 한국어로 풀고, 스킬명·명령·경로·산출물 필드처럼 실제 식별자인 영어만 남겼다. 재검토에서 구조 변경 허용 범위, 인라인화 선택지와 동일 입력 검증 범위가 번역 중 좁아진 부분을 찾아 원뜻에 맞게 복구했다.
- 한국어 정리를 포함한 `purpose-fit-design` 최종 커밋 `effc833`과 `structure-first` 최종 커밋 `6996078`을 검증해 원격 저장소에 올렸다. 다른 정본이 전진하지 않았음을 확인한 뒤 잠금 정보, 무결성 값, 생성 스냅샷과 출처 기록을 갱신했다.

- 공개 설명을 다시 검토해 제품 계약과 변경 기록의 구체성은 유지하고, README의 스킬 안내와 플러그인 상세 설명·시작 문구만 제품 전체의 장기 약속에 맞게 넓혔다. 이번 릴리스의 두 스킬 개정이 플러그인 전체 정체성이나 모든 작업의 선행 절차처럼 보이지 않도록 조정했다.
- 사용자 재검토와 과거 `structure-first` 호출 결정 기록을 대조해, 이번 중간안이 스킬을 “구체적인 구조 문제가 이미 드러난 작업”으로 과도하게 축소했음을 확인했다. 기능 구현·버그 수정·리팩터링에서 흐름·책임·조합·부작용과 상태·계약 검증을 다루는 원래 역할을 복원하되, 구조 변경은 선택 사항으로 남기고 기계적 수정·일회성 실험·구조 판단 없는 국소 변경만 제외하도록 제품 계약과 정본을 다시 맞췄다.
